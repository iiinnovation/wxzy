# wxzy 学习小程序设计

状态：Baseline v1
日期：2026-07-22
边界：唯一用户的学习运行面，不承担 PDF 处理

## 1. 设计目的

把当前“卡片列表 + 四档评分”原型升级为能够回答三个问题的个人学习工具：

1. 我今天应该学什么？
2. 我为什么要学这些内容？
3. 我的薄弱点和未来负荷是什么？

小程序围绕短时、专注、可追溯的学习会话设计，不建设文档处理后台、社区或营销页面。

## 2. 产品边界

### 小程序展示

- 唯一 Owner 的登录和学习档案。
- 今日计划、到期复习、新内容和薄弱练习。
- 7 本书、章节和已发布卡片。
- 主动回忆、答案、来源、评分和卡片反馈。
- 学科进度、未来负荷和薄弱主题。

### 小程序不展示

- MinerU/Qwen API、batch ID、OCR 文件和解析日志。
- 本地路径、上传签名 URL、数据库或服务密钥。
- 原始 PDF 拆分和失败重跑按钮。
- 未通过审核的候选卡，除非未来建立明确隔离的维护模式。

内容控制面只向小程序提供已发布目录和必要的来源摘录。

## 3. 设计原则

### MP-01 今日任务优先

打开应用直接看到可执行计划，不先进入品牌页、功能介绍或复杂总览。

### MP-02 主动回忆优先

答案默认隐藏；页面不在问题旁泄漏答案关键词。先回忆，再反馈。

### MP-03 时间预算优先

计划以预计分钟为主，不只显示卡片数量。用户可以选择今天学习 10、20 或 30 分钟。

### MP-04 来源随手可查

答案页提供短摘录、书籍、章节和页码；核对来源不需要退出当前会话。

### MP-05 建议必须具体

“建议复习”必须说明具体主题、原因和动作，例如“桂枝汤组成连续两次遗漏，建议拆成药味要点”，而不是泛化鼓励。

### MP-06 克制游戏化

可以显示连续学习和阶段完成，但不使用排行榜、惩罚性断签、随机奖励或社交压力。

## 4. 对标机制

| 来源产品 | 吸收机制 | 在 wxzy 中的落点 |
|---|---|---|
| Anki | FSRS、每日上限、leech、未来负荷 | 后端调度、积压、新卡限制、repair |
| RemNote | PDF/笔记与卡片联动 | 卡片来源抽屉、章节上下文 |
| Quizlet | Know/Still learning、多题型自适应 | 书写检查、判断、对比和周测 |
| Duolingo | 短时路径、清晰下一步 | 今日会话和章节学习进度 |
| 墨墨背单词 | 个人遗忘和复习压力 | 未来 7 天负荷、记忆趋势 |
| 不背单词 | 上下文和关联记忆 | 原文摘录、相关方药/证候对比 |

不复制公共牌组、好友、排行榜、订阅墙和内容市场。

## 5. 一级信息架构

TabBar 使用四个入口：

| Tab | 职责 | 不承担 |
|---|---|---|
| 今日 | 计划、开始会话、完成反馈 | 全量统计、内容维护 |
| 学科 | 书籍、章节、加入学习、来源浏览 | OCR/解析任务 |
| 进度 | 负荷、掌握趋势、薄弱点、周测 | 复杂数据后台 |
| 我的 | Owner、目标、时间预算、会话安全、数据管理 | 学习内容列表 |

任务页面包括：登录/引导、学习会话、章节详情、卡片详情、来源抽屉、周测、薄弱点详情和设置编辑。

## 7. 首次使用与登录

### 7.1 登录状态

- `booting`：读取本地 Session。
- `authenticating`：调用 `wx.login` 和后端交换。
- `needs_owner_claim`：第一次绑定 Owner。
- `needs_onboarding`：未设置学习档案。
- `ready`：进入今日页。
- `forbidden`：OpenID 与 Owner 不匹配。
- `offline`：网络不可用，可展示最近只读摘要但不能提交学习。

### 7.2 Onboarding

采用一个短流程，不收集无关信息：

1. 学习目的：日常巩固/考试准备/专项强化。
2. 可选目标日期。
3. 每日可用时间：10/20/30/自定义分钟。
4. 每周学习日。
5. 学科优先级和当前自评。
6. 计划确认。

默认推荐值必须可跳过和后续修改。

## 8. 今日页

### 8.1 首屏结构

1. 日期和简短状态。
2. 今日预计时长。
3. 主操作“开始今日学习”。
4. 计划构成：到期复习、新卡、薄弱练习。
5. 未来负荷或积压提示，仅在需要行动时显示。

不使用大型营销式 Hero。首屏信息密度适合重复使用，一眼看懂今天的任务。

### 8.2 计划状态

- `loading`：稳定骨架，不让按钮跳动。
- `ready`：显示预计分钟和构成。
- `empty`：今日无到期，可选择轻量新内容或休息。
- `overloaded`：显示积压和“今天只复习，不加新卡”。
- `completed`：显示本次结果和下次计划，不继续制造待办焦虑。
- `error`：保留上次摘要，并提供重试。

### 8.3 调整今日时间

使用分段控制或步进器选择 10/20/30 分钟。修改只重新生成当日未开始部分，不改历史和已完成项。

## 9. 学习会话

### 9.1 会话类型

| 类型 | 用途 |
|---|---|
| review | 标准到期复习 |
| learn | 新章节首次学习 |
| repair | 反复失败卡修复 |
| mixed_test | 周度跨主题测试 |

同一次会话可以由后端计划组合任务，但界面始终一次只呈现一个明确动作。

### 9.2 会话顶部

- 退出按钮。
- 当前阶段，如“到期复习”。
- 固定宽度进度条和 `已完成/总任务`。
- 不展示会诱导抢速度的倒计时；内部记录耗时即可。

### 9.3 主动回忆卡

问题区显示：

- 书籍/章节的轻量上下文。
- 问题。
- 可选“写下答案”输入区，仅在卡片配置需要时出现。
- “显示答案”主按钮。

显示答案后：

- 参考答案。
- 分点 answer_points。
- 用户书写答案与要点差异（有书写时）。
- 来源入口。
- Again/Hard/Good/Easy 四档评分。

评分按钮使用中文主标签和简短含义，不依赖用户理解英文 FSRS 术语。

### 9.4 评分语义

| 评分 | 用户语义 |
|---|---|
| Again | 没想起来或主要事实错误 |
| Hard | 想起一部分，但遗漏明显或很吃力 |
| Good | 主要要点正确，正常回忆 |
| Easy | 快速、完整、确定地回答 |

系统不能仅靠颜色区分四档；文字和布局也要明确。

### 9.5 提交行为

- 点击后立即禁用评分区。
- 请求携带 client_attempt_id。
- 超时后允许重试同一个 ID。
- 服务端返回重复提交时按成功处理。
- 成功后再进入下一张，不能乐观更新导致丢记录。

## 10. 新内容学习

新卡不能直接像旧卡一样冷启动评分。首次学习建议：

1. 展示短来源和知识目标。
2. 用户阅读后关闭内容或进入回忆面。
3. 用户用自己的话复述。
4. 显示标准要点和来源。
5. 完成第一次评分，创建 FSRS 状态。

章节首次学习一次只引入少量卡，数量由后端计划控制。

## 11. Repair 流程

反复失败不只是继续显示原卡：

### 触发示例

- 同一张卡多次 Again。
- response_ms 长且评分持续 Hard。
- 同标签多张卡交替错误。
- 用户反馈“过大/表述不清/来源有误”。

### 可选动作

- 返回原文重新阅读。
- 把一张长卡拆成多个最小问题。
- 生成相似方剂/药物/证候对比。
- 安排一次无提示书写。
- 暂停错误卡并创建内容 Issue。

模型生成修复卡仍是候选，不能在当前会话中未经审核直接进入正式复习。

## 12. 来源抽屉

来源从答案区展开，不使用嵌套卡片：

- 书籍和内容版本。
- 章节路径。
- PDF 文件页和印刷页。
- 覆盖答案的短原文摘录。
- 多来源时按引用顺序分组。
- “来源有误”反馈入口。

长原文支持换行、滚动和复制；默认只加载当前卡需要的摘录。

## 13. 学科页

### 13.1 书籍列表

每本书展示：

- 名称和章节数。
- 内容覆盖状态。
- 已发布卡数。
- 已加入学习数。
- 当前进度。

不得把“已处理 100%”显示为“已掌握 100%”。

### 13.2 章节页

- 章节树和顺序。
- 已发布卡数与风险审核状态摘要。
- 加入学习/暂停操作。
- 已学习、待引入、到期和薄弱数量。
- 可浏览发布卡和来源。

加入整章时提示系统会逐日引入，不显示为“一次加入 N 张今日任务”。

### 13.3 搜索

支持书名、章节、问题、答案和标签；结果按发布内容搜索，不搜索 raw OCR。搜索结果按需分页。

## 14. 进度页

### 14.1 核心指标

- 本周实际学习分钟。
- 复习和新卡数量。
- 当前积压。
- 未来 7 天预计分钟。
- 各学科 active enrollment 和掌握趋势。

### 14.2 薄弱点

每个薄弱点显示：主题、证据、最近错误、建议动作和来源书籍。排序考虑重复遗忘、目标优先级和最近发生时间。

### 14.3 周测

- 从已学习内容中跨章节抽题。
- 以书写、判断、对比或结构化回答为主。
- 周测结果单独用于验证掌握，不直接用一次分数重排全部 FSRS。

### 14.4 图表约束

图表只用于趋势和负荷，不堆叠装饰性仪表盘。颜色、数值和说明同时表达，支持空数据和样本不足状态。

## 15. 我的页

- Owner 昵称和登录状态。
- 学习目标、目标日期、每日分钟和每周学习日。
- 学科优先级。
- desired retention 高级设置，默认隐藏在高级区域。
- Session/设备管理。
- 数据导出、备份状态和删除入口。
- 开发模式 API 地址/Token 只在 dev build 显示。

生产版不要求用户手填 Token。

## 16. 客户端状态模型

### 全局最小状态

- `authState`。
- `sessionToken`（只保存在 storage，globalData 可保存运行时引用）。
- `userSummary`。
- `environment`。

今日计划、书籍列表、会话和统计保留在各页面实例，不把大对象堆入 globalData。

### 请求生命周期

- 页面 `onShow/onLoad` 创建 request sequence。
- 新请求使旧 sequence 失效。
- 页面卸载后忽略迟到结果。
- 写请求有提交锁和 client ID。

## 17. API Client 设计

建议拆分：

```text
services/http.js          # request、Session、超时、错误码
services/auth-api.js
services/profile-api.js
services/catalog-api.js
services/learning-api.js
services/insights-api.js
```

`http.js` 负责：

- Base URL 环境选择。
- Bearer Session。
- request_id 透传。
- 401 刷新或重新登录。
- 统一业务错误到用户文案。
- 超时和可重试标记。

页面不得自行拼 Authorization header。

## 18. 通用组件

只在真实重复出现时提取组件：

- `app-header`：紧凑标题和状态。
- `plan-summary`：时间和任务构成。
- `progress-bar`：稳定尺寸进度。
- `source-drawer`：来源展示。
- `error-state`、`empty-state`、`loading-state`。
- `rating-control`：四档评分和提交锁。
- `subject-row`、`chapter-row`：目录行。

页面区块保持未框定布局；卡片只用于重复内容项或真正独立工具，不嵌套卡片。

## 19. 视觉系统

沿用安静、工作型的学习界面：

- 背景：浅中性色。
- 主文本：近黑。
- 主操作：墨绿/青绿，但不让全页面只有一个色系。
- Hard/提醒：琥珀色。
- 错误/Again：克制红色。
- 来源/信息：中性蓝灰或石墨色。

要求：

- 不使用依赖 `:root`/CSS 自定义属性的关键色值。
- 不使用渐变球、装饰性光斑或营销 Hero。
- 卡片圆角不超过 8px，除非微信原生控件要求。
- 图标优先使用现有图标资源或熟悉符号，并提供可访问文本。
- 字体大小不随 viewport 宽度缩放。
- 固定格式控件使用稳定宽高，动态内容不推动关键按钮跳动。

## 20. 文案规范

- 使用“学习、复习、来源、待核对”，不使用“诊断、治疗建议”。
- 错误文案说明用户下一步，不显示堆栈或内部路径。
- 计划原因写具体事实，如“今天到期 12 张”，不写“AI 为你精心安排”。
- 样本不足时写“数据还不够”，不伪造个性化结论。

## 21. 页面状态矩阵

| 页面 | Loading | Empty | Error | Unauthorized | Completed |
|---|---|---|---|---|---|
| 今日 | 计划骨架 | 可选轻量学习 | 重试/上次摘要 | 重新登录 | 会话总结 |
| 学科 | 列表骨架 | 尚无发布内容 | 重试 | 重新登录 | 不适用 |
| 章节 | 章节骨架 | 无可学习卡 | 重试 | 重新登录 | 已加入提示 |
| 会话 | 预取下一题 | 无任务 | 同 ID 重试 | 保存状态后登录 | 本轮结果 |
| 进度 | 指标骨架 | 数据不足 | 重试 | 重新登录 | 不适用 |
| 我的 | 档案骨架 | 未完成引导 | 重试 | 登录页 | 保存成功 |

## 22. 性能设计

- 今日首屏只请求计划摘要和最多 3–5 个预览项。
- 会话一次获取当前任务和可选下一任务，不加载整个大队列。
- 章节和卡片分页。
- 来源正文按展开加载。
- `setData` 只发送变化字段，不反复设置完整 queue。
- 图片使用合理尺寸和失败占位。
- 包体接近阈值时再分包，首阶段保持结构简单。

## 23. 网络和离线

### 开发者工具

允许 `http://127.0.0.1`，关闭合法域名校验仅用于开发。

### 真机开发

使用局域网 IP 或测试 HTTPS 域名；`127.0.0.1` 指向手机自身，不能使用。

### 生产

只使用备案 HTTPS 域名。可以缓存只读目录摘要，但评分和计划变更必须服务端确认后完成。

## 24. 埋点与可观察性

仅记录学习所需事件，不采集输入的完整医学文本：

- onboarding_completed。
- daily_plan_loaded/adjusted。
- study_session_started/completed/interrupted。
- card_revealed。
- review_submitted/retried。
- source_opened。
- card_issue_created。
- chapter_enrolled/suspended。
- weekly_test_completed。

事件包含内部 user_id、card ID、duration、结果和 request_id，不包含 Token、OpenID 或完整答案原文。

## 25. 可访问性与兼容

- 交互目标适合触控，避免过小文字链接。
- 状态不能只靠颜色。
- 长标题和古文换行，不截断核心答案。
- 键盘弹出时输入和提交按钮不互相遮挡。
- 验证当前基础库与最低支持版本。
- 加载、点击和动态文字不能改变固定工具栏尺寸。

## 26. 当前页面迁移

| 当前页面 | 目标 |
|---|---|
| `today` | 保留路由，升级为 DailyPlan 驱动 |
| `review` | 演进为 `study-session`，支持多任务类型和幂等提交 |
| `library` | 演进为 `subjects` + `chapter`，区分发布/加入/掌握 |
| `me` | 从 API/Token 设置转为 Owner Profile；dev 配置条件显示 |
| 无 | 新增 `insights` Tab 和 onboarding |

迁移期间可以保留兼容 API，但新页面只依赖 `/api/v1` 契约。

## 27. 验收场景

1. 新 Owner 登录并在两分钟内完成档案设置。
2. 今日页显示分钟、到期、新卡和薄弱项，开始按钮稳定可见。
3. 用户完成主动回忆、展开来源、评分和下一张。
4. 重复点击或超时重试只产生一个 ReviewAttempt。
5. 用户把一个章节加入学习，当天只出现计划允许的新卡。
6. 学科页明确区分内容覆盖、已发布、已加入和掌握趋势。
7. 进度页能解释未来负荷和具体薄弱点。
8. 生产页面不显示 API Token、MinerU 或本地路径。
9. 长古文、长方名、无来源、空目录、断网和 Session 过期均有完整状态。
10. 开发者工具和真机主要流程无重叠、空白画面和不可见按钮。
